## Design Context

### Users
Primary users are anime fans planning a seichi junrei trip, plus travelers who are already near a station or neighborhood and want to decide whether a detour is worthwhile.

The product supports users in Chinese, Japanese, and English. Their core jobs are:
- discover meaningful scenes from a specific anime
- understand what is worth visiting nearby right now
- turn selected scenes into a realistic half-day or one-day route

### Brand Personality
The interface should feel like a pilgrimage planning studio rather than a generic AI chat app.

Working personality:
- cinematic
- editorial
- dependable

The emotional goal is to reduce blank-page anxiety and replace it with momentum: users should feel oriented, inspired, and ready to go out.

### Aesthetic Direction

Use an editorial, airy visual language (京吹夏季 palette). The interface is a focused
pilgrimage planning studio — never a generic SaaS dashboard.

Design tokens:
- Background: `oklch(98% 0.008 218)` (near-white)
- Foreground: `oklch(20% 0.025 238)` (near-black)
- Card: `oklch(95% 0.012 215)`
- Muted: `oklch(91% 0.016 218)`
- Border: `oklch(85% 0.022 222)`
- Primary (cornflower blue): `oklch(60% 0.148 240)`
- Display font: Shippori Mincho B1 (editorial, Japanese serif)
- Body font: Outfit (clean, modern) + system fallbacks

Strong photography, map textures, and route marks carry visual weight. Typography
leads with the display font for headings and anime titles. The background should
feel like a paper / editorial workspace (map texture, subtle grain), while interaction
states use KyoAni-like color accents. Accents appear sparingly — buttons, active states,
anchor cards — never as a full-page background fill.

Avoid: generic SaaS purple gradients, heavy card stacks, "AI chat assistant" aesthetics,
and over-decorated UI chrome.

### Technical Context

Stack: Next.js 16 (static export), Tailwind v4, base-ui + shadcn/ui, react-leaflet, Supabase auth.
Build: `output: 'export'` → `out/` → Cloudflare Worker ASSETS binding.
Components: ~50 total, CVA-based variants, CSS-variable-aware (no hardcoded Tailwind palette).
Generative UI: registry-based (`registry.ts`) mapping backend intents to visual components.
i18n: ja/zh/en, dictionary-based dynamic imports.

### Layout Direction

**Adaptive hybrid layout** — the interface reshapes based on state, not fixed columns.

- **Chat-focused** (no results yet): chat centered at comfortable reading width (~640px), similar to Claude/ChatGPT. No wasted space.
- **Split** (results arrive): chat narrows to sidebar width, result panel slides in and takes primary focus. Animated transition.
- **Collapsible**: user can collapse result panel to return to chat-focused mode, or collapse chat to go full-screen results/map.
- **Tablet** (768–1024px): two-column without icon sidebar (sidebar becomes overlay or hamburger).
- **Mobile** (<768px): single column + bottom sheet for results (existing pattern, refined).

Reference feel: Claude.ai artifact panel behavior — panels appear/disappear based on content, not fixed grid.
Anti-references: generic SaaS dashboards with permanent empty panels; overly decorated AI chat UIs.

### Accessibility

- WCAG AA target
- Touch targets: 44px minimum (already implemented)
- Reduced motion: `prefers-reduced-motion` (already implemented)
- Outdoor use consideration: sufficient contrast for bright-screen readability
- Focus visible: 2px primary outline with offset (already implemented)

### Design Principles
1. Start from user intent, not from an empty chat box.
2. Keep scenes, map, and route in the same working context.
3. Let imagery and place do narrative work before copy explains it.
4. Treat AI as a route copilot that modifies plans, not as the product's main screen.
5. Make the next action obvious on every screen.
6. The layout serves the content — panels appear when there is content, not before.
7. Every screen size gets a considered experience, not a breakpoint hack.
